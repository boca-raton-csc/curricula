The Problems Facing the Web Today

We will start off the course by talking about challenges that the web faces today.

  • Refrigerators
  • Web vs. Native

Refrigerators

Soon almost everything will be able to access the Web (including your refrigerator). Here are a few things that already can access the web:

  • Computers
  • Phones
  • Cars
  • Watches
  • Televisions
  • Robots
  • Gaming Consoles

...And each device has it's own limitations.

  • Crude input device (Television, Gaming consoles)
  • Tiny screens (Phone, Watch)
  • No screens! (Robots)

The Solution!

As developers we must design content for the web that will work on any device. Known and unknown.

Two fields in the industry that specifically work towards combating this problem are:

Accessibility
Makes sure people with disabilities (like blindness or motor disease) are able to completely access the website. Their work is also useful for devices without a screen which impose a pseudo-blindness.
Responsive Design
Work towards making sure site designs look good at any screen size/resolution/connection speed.

Web vs. Native

There is a huge conversation in the industry over which is better, web or native?

The Web
You beautiful people don't need the same definition twice ;)
Native
Any code written for specific hardware. This is code that only runs on a certain platform. For example, a video game developer must build his or her game for XBox, PlayStation, a Computer, etc.

Standards

In order for the web to work on any hardware, a standards organization called W3 was formed.

This body decides how all the code should work and then each platform (iPhone, Laptop, Refrigerator, XBox) create their own implementation.

So which is better?

Neither. Both platforms have their advantages, and when selecting one it all comes down to what the project needs, and what you prefer.

Native advantages

  • Faster (but arguably marginally so)
  • Opinionated

Web advantages

  • Works anywhere
  • Unopinionated

Internet Explorer

Browsers take our code and render it for the user, there are many different browsers:

  • Chrome
  • Safari
  • Opera
  • Internet Explorer...

Browser Wars

Browsers are always in competition for who can have the largest number of users. It used to be browsers would get more users depending on how many fancy features no one else had. Can you see the problem in that mindset?

There were many problems. Browsers would add new features, but with different syntaxes. Or browsers would add new features and call it something completely different from other browsers.

Ambiguity

One such example is the following. If a developer wrote:

        height: 50px

That code would mean one thing in one browser and a completely different thing in another!

People had to create completely different websites for different browsers. That is a very bad thing considering the goal of the web is to write code that works everywhere.

The Solution

The standards we talked about earlier. The reason no one likes Internet Explorer is because for the longest time they were terrible at implementing the latest standards in a timely fashion. More recently the Internet Explorer team at Microsoft has gotten a lot better at keeping their browser up to date.

Now the winner of the browser wars is the one who can implement new standards the fastest and most accurately. This is great news for developers.
