Refrigerators

Soon almost everything will be able to access the Web (like your refrigerator). Here are a few things that already let you access the web:

  • Computers
  • Phone
  • Car
  • Watch
  • Television
  • Robots
  • Gaming Consoles

The Solution!

Each device that can access the web has it's own limitations.

  • Crude input device (Television, Gaming consoles)
  • Tiny screens (Phone, Watch)
  • No screens! (Robots)
  • On and on and on...

As computer scientists we must design content for the web that will work on any device.

Internet Explorer

The other problem...

Browsers take our code and render it for the user, and there are many different browsers:

  • Chrome
  • Safari
  • Opera
  • Internet Explorer...

Browser Wars

Browsers are always in competition for who can have the larges amount of users. It used to be browsers would get more users depending on how many fancy features they had.

There were many problems with that. Browsers would add new features, but with different syntaxes. Or browsers would add new features and call it something completely different from other browsers.

Ambiguity

One such example is the following. If a developer wrote:

        height: 50px

That code would mean one thing in one browser and a completely different thing in another!

People had to create completely different websites for different browsers.

The Solution

Standards. A group called the W3C created a set of standards defining what features should be implemented by browsers and the exact syntax.

After a lot of work these standards were adopted by all browsers.

Now the winner of the browser wars is the one who can implement new standards the fastest and most accurately. This is great news for developers.
