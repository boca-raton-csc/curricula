A Case for JavaScript

If you have any programming experience you might not be excited to here we will be teaching JavaScript. Here is why we chose JavaScript:

  • Can run no matter the operating system as long as a browser is installed (it's the only programming language of the front end web)
  • Everything is an object
  • Asynchronous concepts very valuable to understand
  • Can emulate syntaxes from many languages (can even create custom syntaxes if macros are implemented in ES7)
  • Weak typed programming languages are typically easier for beginners
  • Constantly being updated (ES6, ES7)
  • Node.js is awesome
  • HUGE community of open source libraries (NPM)
  • It's fun (our opinion)

Most importantly we will be teaching you how to think like a computer scientist. After that go and learn whatever language you want.

Client code

HTML, CSS, and JavaScript are the files that get rendered in the browser, the "client".

Because they are sent to your browser, anyone can see the code you write. Let's go to Google and click inspect element.

View the source with whatever browser you are using, it is completely transparent!

Server code

Because anyone can see your "client" code, it is not secure. Anyone can change any value anytime. The changes just won't be permanent.

People can't see your server side code however.

Server as a Waiter

Imagine the server as your waiter at a resturaunt. The server is what gives you your "cake".

The client (browser) and the server communicate through another web technology HTTP.

Put it all together...

  • I click another cat link (http://cats.cats/cute/kitten)...
  • The browser makes an HTTP request to the Server living at http://cats.cats/cute/kitten
  • The Server sends an HTTP response back to the browser with the HTML of the page
  • The HTML is recieved by the browser and parsed
  • Links to the CSS (http://cats.com/styles.css) and Javascript (http://cats.com/scripts.js) in the HTML are retrieved through more HTTP requests to the Server
  • The CSS is applied to the HTML making the page look pretty
  • The JavaScript is executed on the HTML page, giving it functionality and making the cats go "Meow!"
